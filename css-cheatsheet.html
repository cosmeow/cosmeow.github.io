CSS Cheat Sheet

Reference:

  • https://www.w3schools.com/cssref/css_selectors.asp
  • https://css-tricks.com/almanac/

Cascading Style Sheets at the most basic level it indicates that the order of CSS rules matter.

  • .class: will apply to multiple classes
  • #id: use same id only once, otherwise will break the rules
  • *: sybolizes all elements. not used often. will be overwritten by specificity.
  • element: applies css to specific element
  • element, element: applies css to multiple elements
  • element element: second element tag inside first element tag
  • element > element: first element followed by second element (has a parent element)
  • element + element: any second element immediately after first element
  • :hover: when mouse over, css style shows
  • :last-child: css style will apply to last child
  • :first-child: css style will apply to first
  • !important: not recommended, will override other css selectors ignoring order

What seletors win out in the cascade depends on:

  • Specificity
  • Importance
  • Source Order
